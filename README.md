## What It Does

Eliminates 45 to 60 minutes of manual sprint report writing every sprint. The team fills out a structured Typeform at the end of each sprint. Make picks up the submission automatically, sends it to Claude AI, and Claude generates a complete professional sprint report in under 30 seconds. The report lands in Google Drive ready to share with stakeholders immediately.

## The Problem It Solves

Agile teams spend hours every sprint manually writing reports that summarize what was completed, what was blocked, and what is coming next. The process is repetitive, inconsistent, and pulls time away from actual delivery work.

## How It Works

1. Team fills out a structured intake form — takes approximately 5 minutes
2. Make detects the submission and sends all 17 data points to the Claude API
3. Claude reads the data and generates a complete formatted sprint report
4. Report lands automatically in a designated Google Drive folder

## The Output

The generated report includes:

- Sprint overview table with team, dates, and health status
- Velocity metrics — points planned vs completed with completion rate calculated automatically
- Completed tasks list
- Incomplete tasks with carry-forward sprint noted
- Blockers section with risk assessment
- Bugs and friction log with resolution status
- Next sprint goals
- Stakeholder notes and action items
- Distribution list
- Follow-up meeting flagged at the top if required

## Stack

- Typeform — structured intake form
- Make — workflow orchestration
- Claude API (claude-sonnet-4-6) — AI report generation
- Google Docs — report delivery
